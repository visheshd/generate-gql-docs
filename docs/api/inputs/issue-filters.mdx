---

Ways in which to filter lists of issues.

```graphql
type IssueFilters {
  assignee: String
  createdBy: String
  labels: [String!]
  mentioned: String
  milestone: String
  since: DateTime
  states: [IssueState!]
  viewerSubscribed: Boolean
}
```

### Fields

#### `assignee` ([`String`](/docs/api/scalars/string))

List issues assigned to given name. Pass in `null` for issues with no assigned
user, and `*` for issues assigned to any user.

#### `createdBy` ([`String`](/docs/api/scalars/string))

List issues created by given name.

#### `labels` ([`[String!]`](/docs/api/scalars/string))

List issues where the list of label names exist on the issue.

#### `mentioned` ([`String`](/docs/api/scalars/string))

List issues where the given name is mentioned in the issue.

#### `milestone` ([`String`](/docs/api/scalars/string))

List issues by given milestone argument. If an string representation of an
integer is passed, it should refer to a milestone by its number field. Pass in
`null` for issues with no milestone, and `*` for issues that are assigned to any milestone.

#### `since` ([`DateTime`](/docs/api/scalars/date-time))

List issues that have been updated at or after the given date.

#### `states` ([`[IssueState!]`](/docs/api/enums/issue-state))

List issues filtered by the list of states given.

#### `viewerSubscribed` ([`Boolean`](/docs/api/scalars/boolean))

List issues subscribed to by viewer.
