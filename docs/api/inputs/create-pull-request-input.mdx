### Fields

#### `baseRefName` ([`String!`](/docs/api/scalars/string))

The name of the branch you want your changes pulled into. This should be an existing branch
on the current repository. You cannot update the base branch on a pull request to point
to another repository.

#### `body` ([`String`](/docs/api/scalars/string))

The contents of the pull request.

#### `clientMutationId` ([`String`](/docs/api/scalars/string))

A unique identifier for the client performing the mutation.

#### `draft` ([`Boolean`](/docs/api/scalars/boolean))

Indicates whether this pull request should be a draft.

#### `headRefName` ([`String!`](/docs/api/scalars/string))

The name of the branch where your changes are implemented. For cross-repository pull requests
in the same network, namespace `head_ref_name` with a user like this: `username:branch`.

#### `maintainerCanModify` ([`Boolean`](/docs/api/scalars/boolean))

Indicates whether maintainers can modify the pull request.

#### `repositoryId` ([`ID!`](/docs/api/scalars/id))

The Node ID of the repository.

#### `title` ([`String!`](/docs/api/scalars/string))

The title of the pull request.
